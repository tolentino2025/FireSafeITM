Analise o componente que renderiza as abas “Visão Geral”, “Perfil”, “Formulários”, “Histórico de Relatório”. Sem alterar a lógica nem as rotas, padronize apenas o estilo para corrigir sobreposição e desalinhamento.

No wrapper das abas, adote a classe/estilo .tabs abaixo; em cada link/botão de aba, use .tab. Mantenha o mecanismo atual de “ativo”, mas mapeie-o para .is-active ou aria-selected="true" (sem mudar o código que decide qual aba está ativa).

Aplique este CSS (usando nossas CSS vars --bg, --surface, --text, --border, --ring já definidas):

/* Wrapper das abas */
.tabs{
  display:flex; align-items:center; gap:.5rem; padding:.25rem;
  background:var(--surface); border:1px solid var(--border); border-radius:12px;
  overflow-x:auto; -webkit-overflow-scrolling:touch; scrollbar-width:none;
}
.tabs::-webkit-scrollbar{ display:none; } /* mobile-friendly */

/* Aba (link/botão) */
.tab{
  display:inline-flex; align-items:center; gap:.5rem;
  padding:.625rem 1rem; height:40px;
  border-radius:10px; border:1px solid transparent;
  font-weight:600; font-size:.95rem; line-height:1;
  color:var(--text); background:transparent;
  white-space:nowrap; text-overflow:ellipsis; overflow:hidden;
}

/* Estado ativa */
.tab.is-active, .tab[aria-selected="true"]{
  background:var(--bg); border-color:var(--border);
  box-shadow:0 2px 10px rgba(0,0,0,.08);
}

/* Hover/foco e acessibilidade */
.tab:hover{ background:color-mix(in oklab, var(--text) 6%, transparent); }
.tab:focus-visible{
  outline:none; box-shadow:0 0 0 2px var(--ring); 
}

/* Ícone + texto sempre alinhados */
.tab .icon{ width:18px; height:18px; flex:0 0 18px; }


Garanta que o container das abas receba a classe tabs e cada link/botão de aba receba tab. Preserve <Link/>, onClick etc. Se houver classes utilitárias, apenas adicione tabs/tab (não remova handlers).

Responsivo/mobile: mantenha overflow-x:auto no wrapper; não permitir que o texto quebre linha (usar white-space:nowrap) e não deixar colidir com a aba vizinha (o gap já resolve).

Tema claro/escuro: usar apenas as vars já existentes; não hardcode cores. Verifique contraste AA entre --bg/--surface e --text.

Critérios de aceite:
– Nada de texto sobreposto; ícone e rótulo centralizados na vertical;
– Aba ativa em “pill” clara com borda sutil; inativas com fundo transparente e hover leve;
– No mobile, a barra rola horizontalmente sem quebrar texto;
– Foco visível (teclado) e boa legibilidade em claro/escuro;
– Nenhuma mudança em rotas, eventos ou lógica dos componentes
